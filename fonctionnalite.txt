2ème fonctionnalité : Notes

Le but est que chaque user puisse noter une fois tous les restaurants qu'ils veulent
Les restaurants ont plusieurs notes et ont chacun une moyenne de ces dernières
La moyenne se calculera dans la requête SQL
Il faut donc créer une table Rates qui contient les notes des users pour les restaurants
Les users et les restaurants auront donc une clé étrangère vers cette table

Il faudra donc faire un RateController qui gèrera les routes suivantes :
- /rates/new : permet de créer une nouvelle note
- /rates/edit : permet de modifier une note
- /rates/delete : permet de supprimer une note
- /rates/show : permet d'afficher une note
- /rates/index : permet d'afficher toutes les notes

Ces routes devront être sécurisées (seul un user connecté peut noter un restaurant)
Les users doivent pouvoir créer une note, la modifier et la supprimer. Ils pourront également voir la moyenne du restaurant
Les admins peuvent effectuer toutes les actions sur les notes

La suppression sera un SoftDelete (on ne supprime pas vraiment l'entrée de la base de données, on met juste un booléen is_alive à false pour ne pas l'afficher)

Il y aura aussi un RateRepository qui gérera les requêtes SQL pour les notes

Il y aura aussi le RateEntity qui représentera une note

Il faudra également mettre en place les fixtures pour les notes afin de pouvoir tester le tout
